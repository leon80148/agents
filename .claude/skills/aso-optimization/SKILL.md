# ASO 優化 (App Store Optimization)

此 Skill 提供應用商店優化的專業知識，包括關鍵字研究、元數據優化、轉換率提升和效能追蹤。當需要準備應用商店上架資訊或提升應用程式可見度時，請運用這些策略。

## 各平台 ASO 最佳實踐

### Apple App Store
- 30 字元標題限制（明智使用）
- 副標題：30 字元的關鍵字黃金地帶
- 關鍵字欄位：100 字元（無空格，使用逗號）
- 描述中不要堆砌關鍵字
- 更新可能觸發重新審核

### Google Play Store
- 50 字元標題限制
- 簡短描述：80 字元（對轉換至關重要）
- 長描述中的關鍵字密度很重要
- 可更頻繁地更新
- 平台內建 A/B 測試功能

## 關鍵字研究框架
1. **種子關鍵字**：描述你 app 的核心詞彙
2. **競爭對手分析**：他們排名的關鍵字
3. **搜尋建議**：自動完成的寶藏
4. **相關應用程式**：類似 app 的關鍵字
5. **使用者語言**：他們如何描述問題
6. **趨勢識別**：上升中的搜尋詞

## 標題公式模板
- `[品牌]: [主要關鍵字] & [次要關鍵字]`
- `[主要關鍵字] - [品牌] [價值主張]`
- `[品牌] - [效益] [類別] [關鍵字]`

## 截圖優化策略
1. **第一張截圖**：用主要價值主張吸引注意
2. **第二張**：展示核心功能
3. **第三張**：突顯獨特功能
4. **第四張**：社會認同或成就
5. **第五張**：行動呼籲或效益總結

## 描述結構
```
開場鉤子（前 3 行 - 最重要）：
[引人注目的問題/解決方案陳述]
[主要效益或差異化]
[社會認同或可信度標記]

核心功能（可快速瀏覽的列表）：
• [功能]：[效益]
• [功能]：[效益]

社會認同區塊：
★ "來自滿意使用者的引述" - [來源]
★ [令人印象深刻的指標或成就]

行動呼籲：
[給使用者的明確下一步]
```

## A/B 測試優先順序
1. 應用程式圖示（對轉換影響最大）
2. 第一張截圖
3. 標題/副標題組合
4. 有預覽影片 vs 無預覽影片
5. 截圖順序和說明文字
6. 描述開頭幾行

## 常見 ASO 錯誤
- 忽視競爭對手動態
- 設定後就不管的心態
- 只關注搜尋量，不考慮相關性
- 忽略在地化機會
- 不測試視覺素材
- 關鍵字堆砌（會被懲罰）
- 忽視季節性機會

## 衡量指標
- **關鍵字排名**：目標詞彙的位置
- **曝光度分數**：整體可發現性
- **轉換率**：瀏覽到安裝的比例
- **自然流量提升**：ASO 努力帶來的成長
- **評分趨勢**：星級隨時間的變化
- **評論速度**：每日評論數

## 競爭情報
- 每週追蹤競爭對手更新
- 監控他們的關鍵字變化
- 分析他們的 A/B 測試
- 從他們的評論回覆中學習
- 識別他們的流量來源
- 發現市場機會

## 快速 ASO 優化
1. 在副標題中加入關鍵字（iOS）
2. 優化前 3 張截圖
3. 包含趨勢關鍵字
4. 回覆最近的評論
5. 針對季節相關性進行更新
6. 測試新的應用程式圖示

## 評分與評論管理
- 設計鼓勵正面評論的提示
- 策略性地回覆評論
- 在評論中識別功能需求
- 管理和緩解負面回饋
- 追蹤評分趨勢和影響
- 建立可持續的評論增長速度

## 轉換率優化
- 分析漏斗中使用者的流失點
- 測試不同的價值主張
- 優化「首屏」體驗
- 創造緊迫感但不過於強硬
- 有效突顯社會認同
- 預先解答使用者的疑慮
