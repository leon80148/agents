# 成長駭客 (Growth Hacking)

此 Skill 提供快速用戶獲取、病毒式傳播循環和數據驅動成長實驗的專業知識。當需要設計成長策略或優化用戶獲取時，請運用這些框架和戰術。

## AARRR 框架（海盜指標）
- **A**cquisition（獲取）：讓用戶來到你的產品
- **A**ctivation（啟用）：首次正面體驗
- **R**etention（留存）：讓用戶回來
- **R**eferral（推薦）：用戶推薦給他人
- **R**evenue（營收）：用戶基礎變現

## 成長方程式
```
成長 = (新用戶 × 啟用率 × 留存率 × 推薦率) - 流失
```
- 獨立優化每個變數
- 專注於最高影響力的改進
- 複利效應放大成長

## ICE 優先順序框架
- **I**mpact（影響力）：對成長的潛在效果
- **C**onfidence（信心）：成功的可能性
- **E**ase（容易度）：實施所需的資源
- 為每個實驗評分以進行優先排序

## 病毒式循環藍圖
1. 用戶從產品中獲得價值
2. 產品鼓勵分享
3. 分享的內容吸引新用戶
4. 新用戶進入循環

## 追蹤的關鍵指標
| 類別 | 指標 |
|------|------|
| 獲取 | CAC、管道表現、轉換率 |
| 啟用 | 到達價值的時間、上手完成率、功能採用 |
| 留存 | DAU/MAU、流失率、群組留存曲線 |
| 推薦 | 病毒係數、推薦率、分享率 |
| 營收 | LTV、ARPU、回收期 |

## 成長駭客戰術

### 獲取駭客
- 利用其他平台的成長（平台駭客）
- 創建吸引目標受眾的工具
- 建立 SEO 友善的用戶生成內容
- 實施策略性合作夥伴關係

### 啟用優化
- 減少到達首次價值的時間
- 快速創造「啊哈時刻」
- 個人化上手流程
- 移除摩擦點

### 留存策略
- 建立養成習慣的功能
- 創建互動循環
- 實施挽回流失用戶活動
- 開發社群功能

### 推薦機制
- 激勵分享計劃
- 社會認同整合
- 讓分享對分享者有益
- 減少分享摩擦

## 實驗方法

### 1. 假設形成
- 基於數據見解
- 清晰的成功指標
- 特定的時間範圍
- 可衡量的結果

### 2. 快速測試
- 最小可行測試
- 快速迭代週期
- 多個平行實驗
- 快速失敗/擴展決策

### 3. 數據收集
- 適當的追蹤設置
- 統計顯著性
- 群組分析
- 歸因模型

### 4. 擴展贏家
- 漸進式推出方法
- 資源分配
- 系統建立
- 持續優化

## 管道專屬策略

### 自然管道
- SEO 內容擴展
- 社群媒體病毒性
- 社群建立
- 口碑優化

### 付費管道
- LTV:CAC 優化
- 大規模創意測試
- 受眾擴展策略
- 再行銷優化

### 產品管道
- 產品內推薦
- 網路效應
- 用戶生成內容
- API/整合成長

### 合作夥伴管道
- 策略性整合
- 聯合行銷機會
- 聯盟優化
- 通路合作夥伴

## 成長駭客心態
- 以系統思考，而非戰術
- 數據驅動決策，而非意見
- 學習速度優於完美
- 從第一天就考慮可擴展性
- 用戶價值創造可持續成長
- 在限制中發揮創意
- 快速失敗，學得更快

## 與 6 天衝刺的整合
- **第 1-2 天**：分析與機會識別
- **第 3-4 天**：快速實驗
- **第 5-6 天**：擴展與系統化
