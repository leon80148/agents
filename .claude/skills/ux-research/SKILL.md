# UX 研究方法 (UX Research Methods)

此 Skill 提供用戶研究、行為分析、旅程地圖和可用性測試的專業知識。當需要理解用戶需求、驗證設計決策或分析用戶行為時，請運用這些方法和框架。

## 精簡 UX 研究原則
1. **從小開始**：與 5 個用戶測試比計劃 50 個更好
2. **快速迭代**：多個小研究勝過一個大研究
3. **混合方法**：結合質性和量化數據
4. **務實**：完美的研究延遲交付毫無影響
5. **保持中立**：讓用戶的行為給你驚喜
6. **行動導向**：每個洞察都必須建議下一步

## 快速研究方法工具包
- **5 秒測試**：第一印象分析
- **卡片分類**：資訊架構驗證
- **A/B 測試**：數據驅動決策
- **熱力圖**：了解注意力模式
- **會話錄製**：觀察真實行為
- **離開調查**：了解放棄原因
- **游擊測試**：快速公眾回饋

## 用戶訪談框架
```
1. 暖場 (2 分鐘)
   - 建立融洽關係
   - 設定期望

2. 情境 (5 分鐘)
   - 了解他們的情況
   - 了解替代方案

3. 任務 (15 分鐘)
   - 觀察實際使用
   - 記錄痛點

4. 反思 (5 分鐘)
   - 收集感受
   - 發掘渴望

5. 結束 (3 分鐘)
   - 最後想法
   - 下一步
```

## 旅程地圖元件
- **階段**：認知 → 考慮 → 入門 → 使用 → 擁護
- **行動**：用戶在每個階段做什麼
- **想法**：他們在想什麼
- **情緒**：他們的感受（挫折、愉悅、困惑）
- **接觸點**：他們與產品互動的地方
- **機會**：改善體驗的地方

## 人物誌範本
```
名稱：[易記的名字]
年齡與人口統計：[僅相關細節]
技術熟練度：[對技術的舒適度]
目標：[他們想達成什麼]
挫折：[當前痛點]
行為：[他們如何行動]
偏好功能：[他們重視什麼]
引言：[捕捉他們的本質]
```

## 研究衝刺時程（1 週）
- 第 1 天：定義研究問題
- 第 2 天：招募參與者
- 第 3-4 天：進行研究
- 第 5 天：綜合發現
- 第 6 天：呈現洞察
- 第 7 天：規劃實施

## 要追蹤的分析
- **用戶流程**：用戶去哪裡和在哪裡流失
- **功能採用**：什麼被使用
- **價值實現時間**：用戶多快成功
- **錯誤率**：用戶在哪裡掙扎
- **搜尋查詢**：用戶找不到什麼
- **支援工單**：常見問題

## 可用性指標
- **任務成功率**：用戶能完成目標嗎？
- **任務時間**：需要多長時間？
- **錯誤率**：錯誤發生頻率？
- **學習性**：用戶改進多快？
- **滿意度**：用戶感覺如何？

## 洞察呈現格式
1. **關鍵發現**（一句話）
2. **證據**（數據/引言）
3. **影響**（為什麼重要）
4. **建議**（該做什麼）
5. **工作量**（實施難度）

## 常見研究陷阱
- 引導性問題偏向回應
- 只與團隊成員測試
- 忽視量化數據
- 過度研究次要功能
- 不包含邊緣情況用戶
- 呈現發現時沒有建議

## 遠端研究工具
- **Maze**：快速可用性測試
- **Hotjar**：熱力圖和錄製
- **Typeform**：引人入勝的調查
- **Calendly**：用戶訪談排程
- **Loom**：分享研究發現
- **Miro**：協作旅程繪製

## 研究倫理
- 始終獲得同意
- 保護用戶隱私
- 公平補償
- 對使用透明
- 允許隨時退出
- 安全儲存數據
